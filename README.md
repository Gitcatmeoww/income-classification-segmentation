# Retail Income Targeting and Customer Segmentation

This project analyzes the 1994/95 U.S. Census Bureau Current Population Survey extract for a retail marketing use case. It produces two connected deliverables:

1. A ranked income score that prioritizes individuals likely to earn more than `$50K`.
2. An adult customer segmentation model for campaign planning and message strategy.

The final report is in [report/](report/). The executable analysis is organized as notebooks under [notebooks/](notebooks/).

## Project Structure

```text
data_raw/                 Raw input files, not tracked in git
notebooks/01_eda.ipynb    Exploratory analysis and feature-engineering rationale
notebooks/02_classification.ipynb
                           Income classifier training, validation, testing, and saved model bundle
notebooks/03_segmentation.ipynb
                           Adult customer segmentation, profiling, and saved segmentation bundle
outputs/figures/          Figures generated by the notebooks and used in the report
outputs/models/           Saved classifier and segmentation bundles
report/final_report.md    Final business-facing report source
report/final_report.pdf   Final PDF report
report/compact.css        Print stylesheet used for HTML/PDF export
```

## Environment Setup

This project uses Python `>=3.13` and [`uv`](https://docs.astral.sh/uv/) for environment management.

Install `uv` if needed:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

From the project root, install the pinned dependencies:

```bash
uv sync
```

This creates a local virtual environment at `.venv/` and installs the dependencies pinned in [uv.lock](uv.lock).

Optional shell activation:

```bash
source .venv/bin/activate     # macOS / Linux
.venv\Scripts\activate        # Windows
```

Activation is not required if commands are run through `uv run`.

Place the raw take-home data files in [data_raw/](data_raw/) before running the notebooks:

## Execute the Code

Run the notebooks in order. The order matters because later notebooks consume artifacts from earlier work, especially the classifier bundle used by segmentation.

### Option A: Run Interactively

Start Jupyter Lab:

```bash
uv run jupyter lab
```

Run the notebooks in this order:

1. [notebooks/01_eda.ipynb](notebooks/01_eda.ipynb)
2. [notebooks/02_classification.ipynb](notebooks/02_classification.ipynb)
3. [notebooks/03_segmentation.ipynb](notebooks/03_segmentation.ipynb)

### Option B: Run From Terminal

To re-execute the full analysis without opening Jupyter Lab:

```bash
uv run jupyter nbconvert --to notebook --execute --inplace notebooks/01_eda.ipynb
uv run jupyter nbconvert --to notebook --execute --inplace notebooks/02_classification.ipynb
uv run jupyter nbconvert --to notebook --execute --inplace notebooks/03_segmentation.ipynb
```

The notebooks write generated figures and model artifacts under [outputs/](outputs/).

## Reproducibility Notes

- The notebooks use fixed random seeds where model randomness is involved.
- The classification workflow uses a fixed 60/20/20 train/validation/test split.
- The test set is held out until the final classifier evaluation.
- Census sample weights are used for population-level reporting and evaluation metrics.
- The segmentation notebook clusters adults only (`age >= 18`) and profiles the resulting segments with weighted statistics.
- The segmentation model uses TruncatedSVD dimensionality reduction before K-Means to make distance-based clustering more stable on sparse one-hot encoded features.
- The final classifier and segmentation bundles are small enough to be versioned in this repository; raw data remains local only.

## Generate the Report

The report source is:

```text
report/final_report.md
```

It references generated figures in [outputs/figures/](outputs/figures/), so re-run the notebooks first if those files are missing or stale.

Report export uses `pandoc`. Install it if needed:

```bash
brew install pandoc     # macOS
```

Generate the standard HTML report:

```bash
pandoc report/final_report.md \
  -o report/final_report.html \
  --from markdown \
  --standalone \
  --resource-path=report:outputs/figures
```

Generate the print HTML used for PDF export:

```bash
pandoc report/final_report.md \
  -o report/final_report_compact.html \
  --from markdown \
  --standalone \
  --css compact.css \
  --resource-path=report:outputs/figures
```
